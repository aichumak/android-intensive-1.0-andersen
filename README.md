Краткое описание работы приложения:   
!В приложении есть один баг, который не успел исправить - при первом запуске приложения будут отображаться пустые списки, т.к. в БД сначала будут отсутствовать данные, что бы данные загрузились из сети в БД и отобразились в списках нужно понажимать вкладки (персонажи, локации, экпизоды) главного экрана и подождать пару секунд, после данные загрузятся из сети в БД и отобразятся в списках.  
При первом запуске приложения выполняются запросы в интернет для получения всех необходимых данных из сети, все полученные данные сразу записываются в БД. Пагинация в приложении не реализована. Далее приложение работает только с данными из БД, если в БД данные отсутствуют, все списки будут отображаться пустыми. В приложении не реализована проверка состояния сети, сообщения об отсутствии сети или данных не реализованы. При переходе по вкладкам, при каждом переходе выполняется повторная загрузка данных из сети в БД (обновление), прогресс бар загрузки данных не реализован, в момент загрузки данных будут отображаться пустые списки (или заполненные), после загрузки обновленные данные отобразяться в списках.   
Все изображения хранятся в БД в виде url ссылок и загружаются из сети сразу в списки или на экраны детальной информации. При отсутствии сети, некоторые изображения могут не отображаться на экране.  
Приложение реализовано по архитектуре Single Activity, при запуске первоначально загружается splash активити, затем главное активити. Главное активити содержит главный фрагмент контейнер развернутый на весь экран, далее при работе с приложением в этот контейнер загружаются все необходимые фрагменты. Фрагмент главного экрана содержит панель навигации и вториный контейнер фрагментов, в который загружаются фрагменты выбранной вкладки навигации. В приложении полностью реализована навигация по фрагментам. Также во фрагментах детальной информации реализованы клики по полям вью location и origin для перехода во фрагменты детальной информации.   
На главном экране реализованы фильтры и поиск. Чтобы отменить фильтрацию, нужно повторно открыть экран фильтра и нажать кнопку Cancel. Фильтрация и результы поиска сбрасываются при переходе на другую вкладку.    
Для загрузки данных из сети и обработки результатов используются библиотеки RxJava2, Retrofit2, GSON.    
Для БД использутся библиотека Room.    
Для получения данных из Room используются корутины и suspend функции.
